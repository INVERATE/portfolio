Application d’audit d’ordinateurs

Réalisé en stage chez Revalo

Objectif

L’objectif de cet outil est de permettre d’auditer un lot d'ordinateurs facilement, en limitant au maximum les erreurs humaines et en améliorant la fiabilité des données. Ce projet est divisé en deux parties fonctionnant en coordination :

  • application locale embarquée sur clé USB bootable (Lubuntu) permettant d'effectuer des tests automatisés et manuels sur un PC.
  • site web pouvant recevoir les résultats d'audit à partir des clés USB, centralisant les données et permettant de les consulter, modifier et exporter.

Partie clé USB — application locale

J’ai développé une application métier stockée sur une clé USB bootable (Lubuntu LTS, distribution allégée d'Ubuntu). L’utilisateur branche la clé, boote dessus et l’application démarre automatiquement lorsque l'ordinateur s'allume.

Fonctionnalités principales

  • Tests automatiques : CPU, RAM, stockage, GPU, réseau, batterie, température, etc.
  • Tests manuels : certains tests nécessitent une interaction humaine, comme le clavier, écran , caméra, audio, châssis, ports USB.
  • Envoi automatique vers le serveur central si connexion Internet disponible. Un fichier de log est enregistré sur la clé.

Aspects techniques et contraintes

  • Choix OS : Lubuntu LTS via Rufus (mode Try pour compatibilité avec la majorité des ordinateurs / Secure Boot)
  • Gestion de la persistance sur clé pour conserver paramètres et logs.
  • Duplication : workflow de création d’images (.img) pour cloner et déployer plusieurs clés. Les mises à jour sont effectuées via GitHub avec une clé SSH.

Partie site de gestion

J’ai également conçu un site avec une API REST sécurisée et une base de données centralisée pour recevoir, valider et stocker les audits envoyés par les clés. Ce site web permet de consulter, filtrer, éditer et exporter les données.

Fonctionnalités principales

  • Réception des données à partir des données des clés USB, une fois l'audit confirmé par le technicien. Le site va ensuite envoyer une confirmation de réception sur l'appareil testé pour indiquer qu'il peut être éteint.
  • Authentification API et contrôle d’accès (permissions par rôle).
  • Export : Excel et connexion directe vers l'ERP existant de l'entreprise
